# Local Dependencies (Interop Assemblies)

### Why are these files here?
As this is my first modding project, I am still learning how to properly configure a build pipeline that automatically fetches all dependencies from remote repositories. To ensure the project compiles correctly in GitHub Actions and for other contributors, I have included the necessary library files in this folder.

### Source & Safety
All files in this directory are sourced directly from the original and official locations:
* **BepInEx & Il2CppInterop**: Downloaded from the official [BepInEx Bleeding Edge releases](https://builds.bepinex.dev/projects/bepinex_be).
* **Unity Assemblies**: These are interop (proxy) assemblies generated by the game engine and BepInEx during the first run of *Little Witch Nobeta*.

### Important Note
These `.dll` files do not contain any game logic, copyrighted code, or executable binaries from the game itself. They are metadata-only "stubs" (interop assemblies) required for the C# compiler to understand the game's internal structure.

If you have experience with automated dependency management for IL2CPP projects and would like to help improve this, feel free to open a Pull Request!
